    Single Sign On - SSO

    Single sign-on (SSO) makes it easier to centrally manage access to multiple accounts, web apps, and mobile apps, giving users single sign-on access to all assigned accounts and apps from one place. It permits users to tape one username and password to access multiple applications.

    The projects consist in configuring and administering an access management solution and offering authentication (SSO) and an identity federation for the applications.

    The client

    Small, medium and large businesses.

    Objectives

    Users prompted for credentials once can access multiple applications

    • Save time and effort

    • Less passwords to remember.

    • Prevent security attacks (phishing).

    • Reduce operating costs

    • Access control

    • Granular audit

    • Great user experience

    • Fewer password recovery support tickets

    Challenges

    • Single point of failure. Users will not be able to access the resource if it stops functioning.
    • If an authenticated user is granted access to multiple applications, their credentials may become available to other users for misuse. To better protect your credentials, SSO should be combined with other authentication methods.

    About the SSO Solution

    The goal of this project is to realize the security benefits of user authentication and to strengthen authentication with the help of multi-factors. At the beginning of this project, some analysis from existing documents (architecture documents, studies, etc.). Then, I elaborated a plan which consists in collecting information on each application to be migrated, in analyzing its documentation, in configuring the access management solution and in testing the migration as well as in realizing the documentation of the migration. To carry out this plan, templates are created to define the chosen approach (different phases of the plan), to collect information on the applications to be migrated and to document the different stages of the migration and the tests.

    For some applications recommendation documents are elaborated to propose alternatives solutions, their impacts and recommended solutions. I developed access managment services on the application side to adapt the applications to this solution.
